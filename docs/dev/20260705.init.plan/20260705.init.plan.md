# hdrt 初始化规划

## 1. 项目定位

`hdrt` 是一个跨平台硬件信息 CLI / TUI 工具。

名称含义：

- `hd`: Hardware Device
- `r`: Rust
- `t`: Ratatui

完整解释：

- `hdrt`: Hardware Device Rust Ratatui
- `hard ratatui`: 面向硬件信息查看的 Ratatui 终端工具
- 快速记忆时可以记成 `"hard rata"`，方便联想到简写指令 `hdrt`

核心目标：

- 在终端里快速查看硬件信息。
- 普通 CLI 输出要适合复制、记录、贴进文档。
- TUI 模式要适合交互式查看、切换硬件分类。
- 能显示多少就显示多少；权限不足时不直接失败，而是提示如何获得更完整信息。

第一阶段重点不是替代所有硬件检测工具，而是把常用硬件信息整理成好看的跨平台输出。

## 2. 目标平台

优先支持：

- Windows x86_64 / aarch64
- Linux x86_64 / aarch64
- macOS x86_64 / aarch64
- Android / Termux aarch64 / x86_64

实现顺序：

1. Linux
2. Windows
3. macOS
4. Android / Termux

Linux 是第一优先级，因为 `lsblk`、`smartctl`、`dmidecode`、`lscpu` 这套工具链最适合先做出完整体验。

## 3. 命令设计

基础命令：

```bash
hdrt disk
hdrt mem
hdrt memory
hdrt cpu
hdrt mb
hdrt motherboard
hdrt all
hdrt tui
hdrt doctor
```

别名：

- `mem` 等同于 `memory`
- `mb` 等同于 `motherboard`

推荐参数：

```bash
hdrt disk --format table
hdrt disk --format wide
hdrt disk --format json
hdrt disk --format markdown
hdrt disk --detail basic
hdrt disk --detail smart
hdrt disk --detail full

hdrt mem --format table
hdrt cpu --format json
hdrt all --format markdown
hdrt tui --tab disk
hdrt doctor
```

## 4. 输出格式

计划支持：

- `table`: 默认格式，适合人看。
- `wide`: 更多字段的宽表。
- `compact`: 窄终端友好格式。
- `json`: 给脚本和自动化流程使用。
- `markdown`: 适合复制进文档。
- `csv`: 后续可选。
- `yaml`: 后续可选。

默认输出尽量美观，但不牺牲可读性。

表格输出要正确处理中文、宽字符和窄终端。普通 CLI 表格优先使用成熟库，不手写 `printf` 对齐逻辑。

## 5. 权限策略

原则：

- 能显示多少就显示多少。
- 不因为缺少 root / 管理员权限直接整体失败。
- 无权限字段显示 `Unknown`、`Permission required` 或空值，视字段语义决定。
- 在输出底部给出明确提示。

Linux 示例：

```text
提示:
- SMART 健康、Model Family、坏道计数需要 smartctl，推荐使用 sudo hdrt disk --detail smart
- 内存插槽序列号通常需要 dmidecode，推荐使用 sudo hdrt mem
```

Windows 示例：

```text
提示:
- 部分主板、BIOS、磁盘序列号字段需要管理员 PowerShell
- 推荐以管理员身份运行: hdrt disk --detail full
```

`doctor` 命令负责检查：

- 当前是否 root / 管理员。
- `smartctl` 是否存在。
- `dmidecode` 是否存在。
- `nvme` 是否存在。
- 当前平台支持哪些采集后端。
- 哪些字段可能因为权限不足而缺失。

## 6. 数据采集策略

采集层要区分：

- 基础信息：普通用户权限尽量可读。
- 扩展信息：需要外部命令。
- 敏感/低层信息：可能需要 root / 管理员。

采集失败时返回结构化 warning，而不是只返回字符串错误。

### Linux / Android

磁盘：

- `lsblk --json` 获取设备、型号、序列号、容量、传输总线、旋转标记。
- `smartctl -i/-a/-j` 获取 SMART、Model Family、固件、健康状态。
- `nvme list` / `nvme smart-log` 后续补充 NVMe 细节。

内存：

- `dmidecode -t memory` 获取插槽、容量、频率、制造商、型号、序列号。
- `/proc/meminfo` 作为无权限 fallback，只显示总量和使用情况。

CPU：

- `lscpu --json`
- `/proc/cpuinfo`
- `sysinfo` crate

主板：

- `/sys/class/dmi/id/*`
- `dmidecode -t baseboard`
- `dmidecode -t bios`

Android / Termux：

- 优先 `/proc/cpuinfo`、`/proc/meminfo`、`sysinfo`。
- `lsblk`、`smartctl`、`dmidecode` 大概率缺失，`doctor` 里清楚提示。

### Windows

磁盘：

- CIM / WMI: `Win32_DiskDrive`
- PowerShell fallback
- `smartctl` 可选增强

内存：

- `Win32_PhysicalMemory`

CPU：

- `Win32_Processor`
- `sysinfo` crate

主板：

- `Win32_BaseBoard`
- `Win32_BIOS`

管理员权限不足时保持基础输出，并提示使用管理员 PowerShell。

### macOS

磁盘：

- `diskutil`
- `system_profiler SPStorageDataType`
- `smartctl` 可选增强

内存 / CPU / 主板：

- `sysctl`
- `system_profiler`

## 7. 品牌识别规则

不要随意猜品牌。

磁盘品牌优先级：

1. `smartctl -i` 中的 `Model Family`
2. 平台 API 中明确给出的 manufacturer/vendor
3. 无可靠来源则显示 `Unknown`

不要把型号第一个单词当成品牌。

例如：

- `Seagate Barracuda 7200.12` 可以显示 `Seagate`
- `Western Digital Blue` 可以显示 `Western Digital`
- `AnuceII 96GB` 如果没有可靠品牌来源，应显示 `Unknown`

## 8. Rust 技术栈

核心依赖建议：

```toml
clap = { version = "4", features = ["derive"] }
ratatui = "0.29"
crossterm = "0.28"
sysinfo = "0.32"
serde = { version = "1", features = ["derive"] }
serde_json = "1"
tabled = "0.16"
anyhow = "1"
thiserror = "2"
which = "7"
```

后续可选：

```toml
serde_yaml = "0.9"
csv = "1"
comfy-table = "7"
```

普通 CLI 表格优先 `tabled` 或 `comfy-table`。

`ratatui` 主要用于 `hdrt tui`，不要为了普通一次性输出强行进入 TUI。

## 10. 数据模型草案

核心模型：

- `DiskInfo`
- `MemoryDevice`
- `CpuInfo`
- `MotherboardInfo`
- `HardwareReport`
- `CapabilityReport`
- `Warning`

每个硬件结构里保留：

- 标准字段：跨平台稳定字段。
- 扩展字段：平台相关或工具相关字段。
- warnings：该对象采集时产生的权限/缺工具提示。

## 11. TUI 规划

`hdrt tui` 页面：

```text
[Overview] [Disk] [Memory] [CPU] [Motherboard] [Health] [Warnings]
```

快捷键：

- `Tab` / `Shift+Tab`: 切换页面
- `←` / `→`: 切换页面
- `r`: 刷新采集
- `j` / `k`: 列表上下移动
- `q` / `Esc`: 退出

第一版 TUI 不追求实时监控，先做静态硬件信息面板和手动刷新。

## 13. 分阶段路线

### Phase 1: CLI MVP

- 初始化 Rust 项目。
- 完成 `clap` 命令结构。
- 完成统一数据模型。
- 实现 Linux `disk`、`mem`、`cpu`、`mb` 基础采集。
- 支持 `table`、`json`、`markdown` 输出。
- 实现 `doctor`。

### Phase 2: 权限与增强信息

- 接入 `smartctl`。
- 接入 `dmidecode`。
- 接入 `lsblk --json`。
- 完成权限不足提示。
- 增加 warning 汇总。

### Phase 3: 跨平台基础支持

- Windows CIM / WMI 采集。
- macOS `sysctl` / `system_profiler` 采集。
- Android / Termux fallback。
- 完成 GitHub Actions 多平台编译。

### Phase 4: Ratatui TUI

- 实现 `hdrt tui`。
- 增加 tab 页面。
- 增加 warning 面板。
- 支持手动刷新。

### Phase 5: 分发

- GitHub Release。
- Linux 包。
- Windows 安装方式。
- macOS 安装方式。
- Termux 安装说明。

## 14. 第一版验收标准

Linux 上以下命令可用：

```bash
hdrt disk
hdrt mem
hdrt cpu
hdrt mb
hdrt all
hdrt doctor
hdrt disk --format json
hdrt mem --format markdown
```

无 root 时：

- 不崩溃。
- 能显示基础信息。
- 缺失字段有清楚提示。

有 root 时：

- 能显示更完整的磁盘 SMART 和内存插槽信息。

CI 上：

- Windows / Linux / macOS / Android target 至少能编译通过。
